Date Validation Documentation For use

Default Options:

splitter: "-"

attribute: "placeholder"

beginYear: 1947

endYear: -1

chkExceedsToday: false

difference: [0, 0, 0], // year, month, date

cssForValidDate: "validDate"

cssForInvalidDate: "invalidDate"

Configuration Option Usage   splitter

splitter is the option to supply the symbol to split the date, month, year.

Configuration Option Usage   attribute

attribute is the option to read which attribute to get the date format.

Configuration Option Usage   beginYear

beginYear is the option to inform the library whether the input year is from begin year.

Configuration Option Usage   endYear

endYear is the option to inform the library whether the input year is with in end year.

Configuration Option Usage   chkExceedsToday

chkExceedsToday is the option to inform the library whether the input date exceeds today or not.

Configuration Option Usage   difference

difference is the option to inform the library to check whether the input date is valid with the difference from today.

Configuration Option Usage   cssForValidDate

cssForValidDate is the option to provide the CSS classes to set for any valid dates.

Configuration Option Usage   cssForInvalidDate

cssForInvalidDate is the option to provide the CSS classes to set for any invalid dates.

Global Methods   init

$.DateValidation   init

  • Initializes the DateValidation library.

Global Methods   isLeapYear

$.DateValidation   isLeapYear

  • Validates whether the input is valid leap year.

Global Methods   isValidDate

$.DateValidation   isValidDate

  • Validates whether the input is valid calender date.

Global Methods   isNotExceedNow

$.DateValidation   isNotExceedNow

  • validates whether the input date exceeds today

Global Methods   isDiffFromNowMatched

$.DateValidation   isDiffFromNowMatched

  • Validates by comparing the difference between the input date and the supplied array [yyyy, mm, dd].

Global Methods   validateInputDate

$.DateValidation   validateInputDate

  • Validates the input date based on the configuration options

Global Methods   setStyleAfterValidation

$.DateValidation   setStyleAfterValidation

  • A supplied CSS class is added for any validated dates (valid or invalid).

Global Methods   errorMessage  

Invalid Calender Date   errorMessage.calenderDate

  • Default Message: (also customizable)   You have Entered the Date which is not available in Calender

If the year is not leap year   errorMessage.leapYear

  • Default Message: (also customizable)   The year you have chosen is not a Leap Year

If begin year not within the specified limit   errorMessage.beginYear

  • Default Message: (also customizable)   The Date you entered is lesser than Begin Year

If end year exceeds the specified limit   errorMessage.endYear

  • Default Message: (also customizable)   The Date you entered is greater than End Year

If the date exceeds today   errorMessage.exceedsToday

  • Default Message: (also customizable)   The Date you entered exceeds today

Final validation of complete configuration   errorMessage.finallyOnCondition

  • Default Message: (also customizable)   The Date Entered is invalid based on the configuration
